 리눅스는 여러명이 한번에 사용할 수 있게 다중사용자 시스템으로 이루어져있음
 다중사용자는 여러사람이 하나의 시스템을 같이 사용하기 때문에 좋은 점도 있지만
 보안의 측면에서 굉장히 취약할 수 있고 그러기에 permission을 어떻게 주냐
 같은 issue도 발생하기 때문에 시스템이 복잡해진다.

 다중사용자 관련 명령어
	user's id를 알려줌
	$ id
	 : 내가 누군지 어느 그룹에 속해있는지 등에 대해 알려준다.
	$ who
	 : 현재 접속해 있는 유저가 누군지 알려준다.

 Root(Super) User(모든 권한을 가짐) VS user(사용자)
	$ sudo 
	 : root user의 권한으로 동작하게함. 
	   그러나 이것은 아무나 쓸 수 없고, root user를 쓸 수 
	   있는 일반유저가 따로 있다.
	   root user의 prompt는 $가 아니라 #이다.
	$ su
	 : root user가 되는 법
	$ su - root
	 : 명시적으로 이와같이도 사용할 수 있다.
	# exit
	 : root user에서 빠져나간다.
	$ sudo passwd -u root
	 : 보통 우분투나 몇 리눅스OS 환경에서는 root 사용자를 막아놓는다.
	   위험한 명령어를 수행할 수 있고 이는 심각한 데미지를 초래할 수 
	   있기 때문이다. 
		$ su
		암호 : 
		su: 인증 실패
	   실제로 우분투 환경에서 위 명령어를 실행하고 비밀번호를 입력한
	   뒤의 출력이다. 그럴때 맨 위에 소개된 sudo passwd -u root는
	   root user로의 사용을 허가하는 명령어이다. 
	   -u : unlock
	   -l : lock
	$ cd ~
	 : 현재 사용자의 홈 디렉토리로 이동
	
	root 사용자의 홈 디렉토리는
	/root 이다. 최상위 디렉토리 / 밑에 있는 root란 디렉토리이다.
